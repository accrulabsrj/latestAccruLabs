# AccruLabs.ai Website Redesign - Phased Implementation Plan

## Executive Summary

This plan outlines a comprehensive redesign of AccruLabs.ai website to match or exceed nas.io quality standards. The project is divided into 5 phases with benchmark testing at each stage. **Every page, subpage, and component will be fully responsive with automatic device detection for mobile, tablet, and desktop compatibility.** No compromise on quality, no hallucination, benchmark-level performance at every phase.

**TARGET AUDIENCE**: Entrepreneurs, Corporates & Finance Professionals
**PRIMARY FOCUS**: Making entrepreneurs responsible AI ready
**VALUE PROPOSITION**: "The Applied AI Lab for Finance, Compliance & Governance" - Training and empowering entrepreneurs to build responsible, governance-first AI systems.

## Critical Requirement: Multi-Device Compatibility

**ALL phases must ensure:**
- ✅ **Mobile compatibility** (< 640px): Touch-optimized, stacked layouts, hamburger menu
- ✅ **Tablet compatibility** (640px - 1024px): Hybrid layouts, touch + mouse support
- ✅ **Desktop compatibility** (> 1024px): Full feature set, hover effects, multi-column layouts
- ✅ **Auto-detection**: Device type, screen size, orientation, touch capability
- ✅ **Responsive design**: Mobile-first approach, fluid typography, adaptive spacing
- ✅ **Performance**: Optimized for each device type (mobile: < 3s, tablet: < 2.5s, desktop: < 2s)

## Phase 1: Foundation & Design System (Week 1-2)

### Objectives:
- Establish design system matching nas.io quality
- Create reusable component library with **full responsive support**
- Set up **device detection system**
- Build benchmark testing framework for **all device types**

### Tasks:

1. **Design System Setup**
   - Create `lib/design-system.ts` with:
     - Color palette, typography, spacing tokens
     - **Responsive breakpoints**: mobile (< 640px), tablet (640px - 1024px), desktop (> 1024px)
     - **Mobile-first spacing scale**: Reduced padding/margins for mobile
     - **Responsive typography scale**: Smaller fonts on mobile, larger on desktop
   - Update `tailwind.config.ts`:
     - Custom theme matching nas.io aesthetics
     - **Responsive breakpoints configuration**
     - **Mobile-first utility classes**
   - Create `app/globals.css`:
     - CSS variables for colors, spacing
     - **Responsive base styles**
     - Dark theme with gradient backgrounds
     - **Touch-friendly base styles** (min tap target 44x44px)

2. **Device Detection & Auto-Detection System**
   - Create `lib/device-detection.ts`:
     - Detect device type (mobile, tablet, desktop)
     - Detect screen size and orientation
     - Detect touch capability
     - Browser detection
     - **Server-side detection** for SSR optimization
   - Create `hooks/useDevice.ts` React hook:
     - Client-side device detection
     - Real-time screen size monitoring
     - Orientation change detection
     - **Auto-adjust layouts** based on device
   - Create `hooks/useResponsive.ts`:
     - Breakpoint utilities
     - Conditional rendering helpers
     - Responsive value getters

3. **Component Library (ALL Responsive)**
   Build reusable components in `components/` with **mobile-first responsive design**:
   - `Card.tsx` - Glassmorphism card (responsive padding: p-4 mobile, p-6 tablet, p-8 desktop)
   - `Badge.tsx` - Status badges (scales font size: text-xs mobile, text-sm desktop)
   - `Button.tsx` - Primary, ghost, gradient variants (full-width mobile, auto desktop)
   - `Section.tsx` - Section wrapper (responsive padding and spacing)
   - `Hero.tsx` - Reusable hero (stacked mobile, side-by-side desktop)
   - `Grid.tsx` - Responsive grid (1 col mobile, 2 col tablet, 3-4 col desktop)
   - `Accordion.tsx` - FAQ accordion (touch-friendly on mobile)
   - `Calendar.tsx` - Class schedule (list view mobile, grid view desktop)
   - `InstructorCard.tsx` - Instructor cards (responsive grid)
   - `SocialProof.tsx` - Enhanced with animations (horizontal scroll mobile)
   - `MobileMenu.tsx` - Hamburger menu with slide-out drawer
   - `ResponsiveImage.tsx` - Image component with device-specific srcsets

4. **Responsive Navigation**
   - `Header.tsx`:
     - **Desktop**: Full horizontal navigation menu
     - **Tablet**: Collapsible menu with dropdown
     - **Mobile**: Hamburger menu with slide-out drawer
     - Sticky header with backdrop blur
   - `Footer.tsx`:
     - **Mobile**: Stacked single-column layout
     - **Tablet**: 2-column layout
     - **Desktop**: Multi-column layout with links

5. **Benchmark Tests (ALL Devices)**
   - **Mobile Testing**:
     - Lighthouse performance: 90+ (mobile)
     - Test on real devices: iPhone SE, iPhone 14, Android phones
     - Touch interaction testing
     - Network throttling (3G, 4G)
   - **Tablet Testing**:
     - Lighthouse performance: 90+ (desktop mode)
     - Test on iPad, Android tablets
     - Touch + mouse interaction
   - **Desktop Testing**:
     - Lighthouse performance: 95+
     - Test on Chrome, Firefox, Safari, Edge
     - Hover states, keyboard navigation
   - **Cross-Browser**: Chrome, Firefox, Safari, Edge (mobile & desktop)
   - **Accessibility**: 95+ on all devices
   - **Responsive Design**: 100% - no horizontal scrolling, all content accessible

### Deliverables:
- Design system documentation with responsive guidelines
- Component library with device-specific variants
- Device detection system functional
- Benchmark test results for all device types
- Development environment ready

## Technical Stack

- **Framework**: Next.js 14 (App Router) with SSR/SSG
- **Styling**: Tailwind CSS with custom design system (**mobile-first**)
- **Responsive Design**: Tailwind breakpoints + custom device detection
- **Device Detection**: Custom utility + useDevice hook
- **Animations**: Framer Motion (with reduced motion support)
- **Forms**: React Hook Form + Zod validation (**mobile-optimized**)
- **Icons**: Lucide React or Heroicons (SVG, scalable)
- **Image Optimization**: Next.js Image component with **responsive srcset**
- **Touch Support**: Touch event handlers for mobile interactions
- **Analytics**: (To be decided)
